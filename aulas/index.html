Comunicação ClienteXServer

Voltar

Como funciona: O cliente: Você digita um endereço, tipo www.exemplo.com, ou clica num link. O navegador manda um "pedido" pro servidor perguntando: "Ei, me manda o conteúdo desse site?". O servidor (o lado que guarda o site): O servidor recebe o pedido, procura os arquivos do site (HTML, imagens, etc.) e manda tudo de volta pro navegador. Se for algo mais complexo, tipo um site com dados personalizados, ele pode gerar o conteúdo na hora antes de mandar O cliente de novo: O navegador pega tudo que recebeu (os arquivos) e monta bonitinho pra você ver e interagir. Um exemplo prático: Você quer abrir o Instagram. O navegador manda um pedido pro servidor do Instagram: "Mostra meu feed?". O servidor busca suas informações, tipo fotos e posts, e devolve tudo pro navegador. O navegador organiza isso e você vê seu feed. É basicamente um "pede e devolve" entre o navegador e o servidor. Eles trabalham juntos pra você ter a experiência completa!

Introdução HTML

Voltar

HTML (HyperText Markup Language) é a linguagem básica usada para criar páginas da web. Ele funciona como a "estrutura" de um site, organizando o conteúdo de forma que os navegadores possam entender e exibir. Principais pontos do HTML básico: Estrutura de uma página HTML Todo documento HTML começa com a declaração para dizer ao navegador que é uma página em HTML5. As tags são usadas para marcar diferentes partes do conteúdo, e quase todas vêm em pares: uma de abertura e uma de fechamento.
